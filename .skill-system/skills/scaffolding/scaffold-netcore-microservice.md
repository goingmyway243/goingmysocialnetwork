# SKILL: scaffold-netcore-microservice

## Purpose
Generate a new .NET Core microservice with proper structure, including API controllers, domain models, repositories, and tests.

## Use When
- Creating a new microservice (User, Post, Notification, etc.)
- Setting up service boilerplate
- Ensuring consistent service structure

## Required Inputs
- Service name (e.g., User, Post, Notification)
- Primary domain entities
- Key business operations

## Expected Output
- Multi-project .NET solution
- API project with controllers
- Domain and application layers
- Infrastructure with repository pattern
- Unit and integration test projects
- Docker file configured

## Execution Approach

### Step 1: Create Solution and Projects with Aspire Support
```bash
# Create solution
dotnet new globaljson --sdk-version 10.0.0 --roll-forward latestFeature
dotnet new sln -n GoingMy.{ServiceName}Service

# Create projects
dotnet new classlib -n GoingMy.{ServiceName}.Domain
dotnet new classlib -n GoingMy.{ServiceName}.Application
dotnet new classlib -n GoingMy.{ServiceName}.Infrastructure
dotnet new webapi -n GoingMy.{ServiceName}.API
dotnet new xunit -n GoingMy.{ServiceName}.Tests
dotnet new xunit -n GoingMy.{ServiceName}.IntegrationTests

# Create Aspire service defaults project (for observability/configuration)
dotnet new classlib -n GoingMy.ServiceDefaults

# Add projects to solution
dotnet sln GoingMy.{ServiceName}Service.sln add src/**/*.csproj tests/**/*.csproj

# Add Aspire NuGet packages to API project
cd src/GoingMy.{ServiceName}.API
dotnet add package Aspire.Hosting
dotnet add package Microsoft.Extensions.Hosting.Abstractions
cd ../../..
```

**NuGet Packages for Service:**

### Step 4: Set Up Core Files

**Program.cs Template (Aspire-Ready):**
```csharp
using GoingMy.{ServiceName}.Infrastructure;
using Serilog;
using OpenTelemetry;

var builder = WebApplication.CreateBuilder(args);

// Add Aspire
builder.AddServiceDefaults();

// Add services
builder.Services.AddControllers();
builder.Services.AddScoped(typeof(IRepository<>), typeof(Repository<>));
builder.Services.AddScoped<I{ServiceName}Service, {ServiceName}Service>();

// Database context with Aspire support
builder.Services.AddDbContext<{ServiceName}DbContext>(options =>
    options.UseNpgsql(builder.Configuration.GetConnectionString("{ServiceName}Db"))
);

// Logging with Aspire observability
builder.Logging.AddConsole();
builder.Services.AddOpenTelemetry()
    .WithTracing(tracing => tracing
        .AddAspNetCoreInstrumentation()
        .AddSource("{ServiceName}Service"))
    .WithMetrics(metrics => metrics
        .AddAspNetCoreInstrumentation()
        .AddMeter("{ServiceName}Service"));

// CORS and API explorer
builder.Services.AddCors(options => {
    options.AddPolicy("AllowAll", builder => builder.AllowAnyOrigin().AllowAnyMethod().AllowAnyHeader());
});
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

var app = builder.Build();

// Map health checks (required by Aspire)
app.MapHealthChecks("/api/v1/health");

if (app.Environment.IsDevelopment()) {
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseRouting();
app.UseCors("AllowAll");
app.MapControllers();

app.Run();
```

**ServiceDefaults Configuration:**
```csharp
// GoingMy.ServiceDefaults/Extensions.cs
using Microsoft.Extensions.Hosting;

namespace GoingMy.ServiceDefaults;

public static class ServiceDefaultsExtensions {
    public static IHostApplicationBuilder AddServiceDefaults(this IHostApplicationBuilder builder) {
        // OpenTelemetry for Aspire Dashboard
        builder.Services.AddOpenTelemetry()
            .WithTracing(tracing => tracing.AddAspNetCoreInstrumentation())
            .WithMetrics(metrics => metrics.AddAspNetCoreInstrumentation());

        // Health checks for Aspire orchestration
        builder.Services.AddHealthChecks();

        return builder;
    }
}
```

**Entity Template:**
```csharp
public class {Entity} : BaseEntity {
    public string Name { get; set; }
    // Add domain properties
    
    public static {Entity} Create(string name) {
        if (string.IsNullOrWhiteSpace(name)) 
            throw new ArgumentException("Name required");
        return new {Entity} { Name = name };
    }
}
```

**Controller Template:**
```csharp
[ApiController]
[Route("api/v1/[controller]")]
public class {Entity}Controller : ControllerBase {
    private readonly I{ServiceName}Service _service;
    
    public {Entity}Controller(I{ServiceName}Service service) {
        _service = service;
    }
    
    [HttpGet("{id}")]
    public async Task<IActionResult> GetById(int id) {
        var result = await _service.GetById{Entity}Async(id);
        return Ok(result);
    }
}
```

### Step 5: Register Service in AppHost (Aspire Orchestration)

**In root AppHost/Program.cs:**
```csharp
// Add PostgreSQL and Redis
var postgres = builder.AddPostgres("postgres")
    .WithVolume("postgres-volume", "/var/lib/postgresql/data");

var {serviceName}Db = postgres.AddDatabase("goingmy-{serviceName}-db");
var redis = builder.AddRedis("redis");

// Add this service
var {serviceName}Service = builder.AddProject<Projects.GoingMy_{ServiceName}_API>("{serviceName}-service")
    .WithReference({serviceName}Db)        // Database reference
    .WithReference(redis)                   // Cache reference
    .WithHttpEndpoint(port: 500{N});        // Assign unique port

// Service-to-service references (if communicating with other services)
{serviceName}Service.WithEnvironment("USER_SERVICE_URL", "http://user-service");

await builder.Build().RunAsync();
```

### Step 6: Add Tests with Testcontainers (Testing with Aspire)

For integration tests, use Testcontainers (compatible with Aspire):

```bash
cd tests/GoingMy.{ServiceName}.IntegrationTests
dotnet add package Testcontainers
dotnet add package Testcontainers.PostgreSql
cd ../../
```

**Integration Test Example:**
```csharp
[Collection("Database collection")]
public class {ServiceName}ServiceTests : IAsyncLifetime {
    private readonly PostgreSqlContainer _postgres = 
        new PostgreSqlBuilder()
            .WithDatabase("goingmy-test")
            .WithUsername("postgres")
            .WithPassword("password")
            .Build();

    public async Task InitializeAsync() {
        await _postgres.StartAsync();
        // Run migrations
    }

    public async Task DisposeAsync() {
        await _postgres.StopAsync();
    }

    [Fact]
    public async Task Create{Entity}_WithValidInput_ReturnsSuccess() {
        // Arrange
        var service = new {ServiceName}Service();
        
        // Act
        var result = await service.Create(new CreateRequest());
        
        // Assert
        Assert.NotNull(result);
    }
}
```

## Quality Criteria
- Solution builds without errors (including Aspire projects)
- All dependencies properly configured
- Service registered correctly in AppHost
- Database context properly set up with Aspire references
- Controllers follow REST conventions with health checks
- Base test infrastructure with Testcontainers
- OpenTelemetry configured for Aspire Dashboard observability
- Service appears healthy in Aspire Dashboard

## Verification Checklist (Aspire-Ready)
- [ ] Solution builds: `dotnet build`
- [ ] Tests run: `dotnet test`
- [ ] Service registers in AppHost: `cd AppHost && dotnet run`
- [ ] Service visible in Aspire Dashboard (http://localhost:18888)
- [ ] Health check endpoint responds: `curl http://localhost:500X/api/v1/health`
- [ ] Database migrations generated and applied via Aspire
- [ ] Swagger/OpenAPI accessible
- [ ] Logging configured and visible in Dashboard
- [ ] Entity relationships defined in DbContext
- [ ] Service interfaces defined
- [ ] CORS configured appropriately
- [ ] Service-to-service references configured in AppHost
